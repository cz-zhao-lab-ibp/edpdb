Miscellaneous

Available commands:
  • comment ,
  • CLEAR ,
  • FILE ,
  • HELP ,
  • SETENV and
  • SHDF .

This type of command is designed to make EdPDB more user-friendly and/or to provide the user with help when needed.

comment

Any text-string following an exclamation mark (!) in an input statement will be considered as a comment and is omitted during command interpretation.

Function: Miscellaneous

Syntax:
1) ! comment
2)  command !comment
Note:
1) The first form is called a comment statement, and the other is called commands with comments.
2) A comment statement starting with a single exclamation mark will be echoed from a macro if the echo option is set to larger than one (1) and verbose is larger than zero (0).
3) A comment statement starting with "!!" will never get echoed.
4) The current version does not support a comment  in a piped command.

See also: SETENV and command interpretation

CLEAR

Clear the terminal screen.

Function: Miscellaneous

Syntax:
CLEAR

FILE

Show the file names which are currently used.

Function: Information, Control

Syntax:
1) FILE
2) FILE OUT file_name
3) FILE LOG file_name

Note:
1) The first form lists the currently used files. For the input PDB files, only the names of the first two and the last one will be listed.
2) Using the OUT option, the intermediate result will be redirected to a new scratch file, so that the old scratch file can be saved for other purposes.
3) Using the LOG option, the result that is usually piped to the stdoutput/sys$output will be redirected to a file.

See also: ACCESS, DISTANCE, HELP, READ, SHDF, SORT and @macro_file

Examples:
1) List the currently used files. 

      file
2) Close the old scratch (.out) file, say a.out, and pipe the scratch output to a file called junk.out. Then one can type the old scratch file. 
      file scr junk.out
      system wait type/page a.out
3) Redirect some displayed result to a file tmp.log, and then switch back to the terminal. 
      file log a.log
      ...
      file log tt:    ! for VMS terminal

HELP

List available commands.

Function: Information

Syntax:
1) HELP or ???
2) command ???

Note: The first form lists the current available commands, and the second form outputs the syntax of a given command. 

See also: FILE, SETENV VERBOSE and SHDF

Examples:
1) Get help on the zone command.

     help
     zone ???

SETENV

Change environmental variables.

Function: Definition, Information

Syntax:
1) SETENV
2) SETENV variable [ new_variables]
3) SETENV -s [variable [ new_variables]]
4) SETENV -r [variable]

Note:
The first form shows the current values.
The second form sets new value(s) or to the default.
The third form saves the current value(s) and sets new values if given.
The fourth one restores the saved value(s).

Available variables include the following:

  • DELIMITER ,
  • ECHO ,
  • EDP_DATA ,
  • INTERACTIVE ,
  • MAXERR ,
  • PROMPT ,
  • TOLOWER ,
  • VERBOSE ,
  • WILDCARD and
  • WINDOW_SIZE .

See also: SET and defining.

DELIMITER

Set the parameter delimiter.

Syntax:
SETENV DELIMITER [one_character]

Note:
The default delimiter is a single quotation mark (').

Examples:
1) Assume that we have a macro file, read.edp, which accepts a text string as its parameter. We can use the delimiter to enclose a text string containing spaces or commas as a single parameter, which otherwise would be considered as multiple parameters. 

      @read 'this is a test.'
2) If a quotation mark (') needs to be included in the text string, the delimiter has to be changed to something else. 
      setenv delimiter #
      @read #it's a test.#

ECHO

Turn the echo option ON/OFF for input commands from a macro file.

Syntax:
SETENV ECHO [0,1,2]

Note:
The default option is 1.

Examples:
1) Turn the echo option off for the input from a macro. 

      setenv echo 0
      @a_well_tested_macro
2) Turn the echo option on for the input from a macro only. 
      setenv echo 1
      @a_macro_to_be_tested
3) Turn the echo option on for the input from both a macro and the terminal, useful for testing commands. 
      setenv echo 2
      @a_macro_to_be_tested

EDP_DATA

A user may setup his/her owner directory to keep the .edp macro files. By default, EdPDB reads an .edp file from the predefined directory ($edp_data/ in unix), if it can not be found in the user's current directory. However, a user may change back and forth between the program-default and one's own default directory using this edp_data option. A variable named edp_data is automatically defined too. 

Syntax:
SETENV EDP_DATA directory_name

Example: 

      setenv edp_data /urs/zhangc/my_project/
           ! set default to the user specified directory
      @task.edp
      ...
      setenv edp_data
           ! back to the program default directory
      @task.edp
      ...

INTERACTIVE

EdPDB can run interactively or batch mode. The program will check the current working mode automatically, and use it as the default. To speed up the calculation, an interactive user may switch between the interactive and non-interactive modes. By doing this, the program will turn on/off LIST and PAUSE, as well as some screen outputs.

Syntax:
1) SETENV INTERACTIVE
2) SETENV INTERACTIVE ON
3) SETENV INTERACTIVE OFF

Note:
The first form sets the mode to the default, without typing the scratch file on the screen. The second form sets the mode to the default, and will type the scratch file on the screen if the default mode is interactive. The third form sets the mode to 'non-interactive'. The output mode is set to off in piped commands.

Examples:
1) Displaying the scratch file can be time consuming, especially when it contains a large number of records. Therefore, the user may deactivate the display and read the file later. 

      setenv interactive off   ! deactivate the display
      ...
      setenv interactive on    ! type the .out file.
      ...

MAXERR

Set the maximum tolerable number of errors.

Syntax:
SETENV MAXERR max_err [(EXIT, QUIT)]

Note:
1) This command sets the value of the maximum tolerable number of errors (max_err) and initializes the number of currently accumulated errors to zero. The initial default is that max_err=1024.
3) If the option EXIT is used, the program will return to the upper level (e.g. macro) when the number of errors is greater than max_err. If the option QUIT is used, the program will quit in the case that the number of errors becomes greater than max_err. The default option is QUIT.
4) To prevent an infinite loop in a macro, the number of accumulated errors increases by one whenever a REWIND GOTO command (with blank option) is issued.
5) To prevent infinite ALIAS substitutions, the number of errors increases by one for each substitution.

Examples:
1) Show the current allowed maximum number of errors. 

      setenv
2) Set the maximum number of errors to 32 and choose EXIT option. 
      setenv maxerr 32 exit

PROMPT

One may reset the prompt, which consists of up to nine characters.

Syntax:
SETENV PROMPT prompt_character_string

Examples:
1) Set the prompt to the text string CHK> followed by a space. 

      setenv tolower off
      setenv prompt 'CHK> '
2) Use the prompt to send messages, eg. from a macro file. 
      setenv prompt 'ok_1> '

TOLOWER

By default, EdPDB converts all the characters in an input statement to lower case before interpreting the commands. All the build-in keyword are in lower case. The user can change the default so that the program will not perform the conversion to the input statement. build-in keyword are in lower case. The user can change the default so that the program will not perform the conversion to the input statement.

Syntax:
SETENV TOLOWER (ON, OFF)

Examples:
1) In the unix system, a file name is case sensitive. To read a file named, eg. UPPER_CASE.FILE, the tolower parameter should be set to off. 

      setenv tolower off
      read UPPER_CASE.FILE
      setenv tolower on

VERBOSE

This option sets output to different levels so that some diagnastic information can be output if necessary.

Syntax:
SETENV VERBOSE level

Note:
The level can be set between 0 and 12; the higher the level is, the more information may be output. The default level is 1 (one). In the output, a string of "command-I6>", for exmaple, is followed by information on the "command" at verbose-level 6, which can be seen when VERBOSE is set to 6 or higher.

Examples:
1) List detailed results of a MATCH3D calculation. 

      setenv verbose 6
      match3d a 4 4.0

WILDCARD

Set the wildcard, which is a single character and is used for example in the SHDF and ATOM commands.

Syntax:
SETENV WILDCARD [one_character]

Note:
The default wildcard is an asterisk (*).

Examples:
1) Assume that, in the input PDB file, one of the atom name is O5*. If we want to select this specific atom type and do not mix it with other atom types such as O5 or O5A, the default wildcard (*) must be changed. 

      setenv wildcard & %
      atom O5*

WINDOW_SIZE

Set the number of lines per window. Because the unix version gets the value from the system directly, this option is not used.

Syntax:
SETENV WINDOW_SIZE [number_of_lines_per_window]

Examples:
1) Set the window size to 40 lines. 

      setenv window_size 40
2) Set the window size to the default value. 
      setenv window_size

SHDF

Show the values defined by DFXXXX command.

Function: Information

Syntax:
SHDF [(*, MAIN, CA, AB, ABC, ABCD, BRG, RES, NEWXYZ, CELL, SYMM)]

Note:
Dfxxxx series commands without following any parameters will display the corresponding definition too.

See also: FILE and SETENV

Examples:
1) To show the status of the ON/OFF switch 

      shdf
2) To show the current definitions, use the wildcard (which is an asterisk `*' by default). 
      shdf *
3) To show the current definition of the torsion angle 
      shdf abcd
